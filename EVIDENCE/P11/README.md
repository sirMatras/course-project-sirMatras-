# P11 — Static Application Security Testing (SAST)

Артефакты формируются в CI workflow для статического анализа безопасности кода.

## Инструменты

### Bandit (Python SAST)
- **Назначение:** Поиск типичных проблем безопасности в Python коде
- **Отчёт:** JSON формат для программной обработки

### Semgrep (Code Analysis)
- **Назначение:** Поиск уязвимостей и проблем безопасности с помощью правил
- **Отчёт:** JSON формат с findings

## Артефакты

- `bandit_report.json` — отчёт Bandit по Python коду
- `semgrep_report.json` — отчёт Semgrep по коду
- `sast_summary.md` — сводка по найденным проблемам безопасности

## Локальный запуск (опционально)

### Bandit

```bash
mkdir -p EVIDENCE/P11

bandit -r app/ -f json -o EVIDENCE/P11/bandit_report.json
```

### Semgrep

```bash
semgrep --config=auto --json -o EVIDENCE/P11/semgrep_report.json app/
```

## Интерпретация результатов

### Bandit
- **HIGH:** Критические проблемы безопасности, требуют исправления
- **MEDIUM:** Важные проблемы, рекомендуется исправить
- **LOW:** Информационные сообщения

### Semgrep
- **ERROR:** Критические проблемы
- **WARNING:** Важные проблемы
- **INFO:** Информационные сообщения

## Использование результатов

Результаты P11 используются для:
- Выявления уязвимостей в коде на ранних этапах
- Валидации security controls
- Планирования улучшений безопасности кода
- Трассируемости от кода к реальным улучшениям
